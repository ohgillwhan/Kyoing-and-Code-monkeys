# 명명 패턴보다 애너테이션을 사용하라

## 핵심 정리

```
애터네이션으로 할 수있는 일들을 명명 패턴으로 처리할 이유는 없다.
자바프로그래머라면 예외 없이 자바가 제공하는 애너테이션 타입들을 사용해야 한다.
```

## 나의 생각

```markdown
명명패턴보다 애너테이션을 사용해야 한다는것을 보자마자 들었던 생각은 두 가지 정도인거같다.
첫번째는 디스패쳐서블릿과 리퀘스트매핑이다.
HttpServlet을 상속받아서 MVC1을 구현한 코드를 보면 execute나 action이라는 메소드명을 강제해야한다.
이 메소드 사용 혹은 명명 강제성을 유연하게 풀어줄수있는 방법은 애너테이션을 이용하고 HttpServlet을 상속하는 디스패쳐에서 애너테이션에 기술해 놓은 input 정보를 이용해 매핑해주는 방법이다. 이걸 이용하면 상당히 유연하게 프로그래밍을 할 수 있다.

두번째는 행동에 의거한 공통로그수집에 용이하겠다라는 생각이다.
회사의 프라이빗 클라우드 솔루션의 로그중에 행동수집 기능은 특정 메소드명에 특정한 이름이 있어야만 한다. 이는 구현하기는 간단했겠지만, 명명패턴이고 하드코딩이기 때문에 제약이 걸리고 무엇보다 안좋은점은 메소드명을 지을때 해당 로그에 대한 이해가 없다면 로그시스템이 해당 메소드의 행동수집을 하지 않을 것이라는 점이다.
이 부분 애너테이션을 이용해서 고도화 해봐야겠다.

다만, 걸리점하나는 애너테이션으로 해결하면 메소드명칭을 자유롭게 할 수 있다는 자유도는 높아지지만, 여전히 어노테이션을 선언하지 않았을때에는 행동수집하지 않을 것이고 이 문제를 compile 단계에서 잡아낼수없다는 점이다.
```
